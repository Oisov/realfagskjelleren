Changes from design

Through the project, we've worked close to the designs we made in P2. However, some changes has been made

Minor changes

The following changes were made to improve user experience and readability.

  • Content width 50% instead of 60%
  • Replaced mail address in footer with a mail icon
  • Reused the footer in mobile/small screen view
  • Changed dropdown menu in mobile/small screen view from a horizontal list to a vertical list
  • h1 size 32px down from original 40px
  • font-size for body increased from 14px to 16px
  • Kept the black background for mobile/small screen view

Structural changes

When we wrote the design plans in P2, we underestimated the amount of text we would produce. To maintain readability of code and css, we split the code and css into several files. We used a global css file to style elements in use in several pages. Css changes that only applied to one page, were kept in local css files. Similarily, fetching events was done on two html pages. Therefore, event methods was seperated into one file.

  • Combined the three javascripts for games into games.js
  • Split events.js into eventMethods.js, facebookEventList and facebookLastEventTimer.js
  • thumbnail.js changed to media.js
  • Went from a single planned css file to several as the main css file got too full
  • Changed name of contact.html to about.html for better correlation to content
  • Added header.js for modularity
  • Renamed thumbnail.js to media.js

    • Made navbar stick to top when scrolling down page
    • Added list of past events in events.html
    • Added detection for mobile devices so that both small screen sizes and mobile devices triggers design for smaller screens

    Modularity

    All the webpages are built up with the same framework. To avoid cluttering of html pages, the footer and header of the pages are removed from the html, and added using javascript. footer.js and header.js contains the javascript used for this, and the html pages simply includes those scripts.
