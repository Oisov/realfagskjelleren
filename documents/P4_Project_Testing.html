Testing Document

Google Forms: Evaluation of Realfagskjellerens website

We based our query several other google forms from previous students and added our own touch. The google form is split into three parts. The first part is to obtain general information about the user, age, browser and operating system.


The second part of the form asks the user to perform some simple tasks using the site. These tasks are meant to mimic real life tasks user might use the site for. Like figuring out how to get to Realfagskjelleren, what is the next event, and so forth.


The last part is a summary of how the user experience was using the site. How easy the navigation was, thoughts on the layout, apperance. Lastly the user was given the option to write their own opinions on what they liked about the page, what they disliked and if they had any ideas on additions to the site.

In the live testing the user was asked to both navigate the site using mobile and a computer if possible. As we had spent a fair bit of time making sure our site was mobile-friendly.

Tester Audience

We wanted both feedback from users well aquainted with Realfagskjelleren and new users. In order to obtain feedback from experiences users we posted the google form query on Realfagskjellerens facebook page.


The live testing was done with users oblivious to Realfagskjelleren. To create a comfortable environment for the user the testing was done with only the user and tester present. We also made sure to inform the user that it was how well the site performed and not the user that was being tested. In addition the test was performed in a calm enviroment to avoid stress for the user. We measured how many missclicks the user performed, and how easilly they were able to answer the questions in the google form.


The users that livetested was choosen in the age range 18-24 and students to create a representative selection.

Testing Platforms

Interestingly enough nearly all of our users used the Chrome/Chromium browser. This made it difficult to discover the bugs related to other browsers. We will come back to this later

In contrast to browsers the varity in operating systems were much broader. Interestingly enough while some users use iOS and OSX they still opt to use Google Chrome as their browsers and not the more common option Safari.

Feedback summary

Scenario 1

We wanted to see how easy this information could be obtained from our site. the premise for this scenario is that the user imagines being a new student looking for information about Realfagskjelleren. The questions also tells a story where the imagined student ends up going to the next event Realfagskjelleren is hosting. This sets the setting nicely for the next scenario.


  • What is Realfagskjelleren?

All answers given in the Google form were correct to this question. Some users noted that it was not clear enough from the text on the "About us" page that all students were welcome, not just those studying STEM (Science, Technology, Engineering and Math).


  • Realfagskjelleren is run by 4 different student unions, which?

The answer to this question was written at the start of the second paragraph on the "About us" page. Here we had two joksters writing something completely wrong. Ignoring those two everyone was able to quickly answer this question correctly. The livetests had no missclicks but complained about the slightly small text.


  • Which event is the next Realfagskjelleren is hosting?

It must be noted that throughout the survey the answer to this question changed, as our home page fetches data from Realfagskjellerens facebook page. Taking this into consideration about 81% of the users answered this question correct. One live tester expressed some confusion as there were no future events, however he was quickly able to deduce this from the text on the frontpage and the empty "Future Events" list on the "Events" page.


  • How easy was it to use the page to naviagate through the questions above?

none of the live testers expressed any difficuly in answering these questions, and the google form mostly confirms this. most users 62% found the questions very easy to answer. about 25% found them easy to answer, while only 6.3% found them difficult.


Scenario 2

In this scenario we wanted to illustrate uses more seasoned students might have for the site and to motivate the users we added an humorstic touch. The premise for this scenario is that the user attended Realfagskjellerens previous event, but by mysterious forces can not seem to remember much, and has lost several important possesions.


  • You attended Realfagskjellerens last event. What is the name of the event you attended?

About 40% of the users was able correctly answer this question correctly. This accounts for both spelling mistakes and that the previous event changes as Realfagskjelleren hosts events.


  • How many other guests attended the same event?

The purpose of this question were to test whether the users realised that the event images were links to the facebook event. This questions was the most difficult to answer nd only about 14% of the users were able to find the correct number of attendees.


Several users where quickly able to quickly able to navigate to the relevant event. After some searching on the page all of them tried to click on the image, taking them to the event on facebook. However due to a strange pagedesign the testers struggled finding the information on facebook. At most this took 4 minutes and 5 misslicks to find the correct number.


  • You barely remember jello shots? Are there any image evidence for this on the website?

In this question we wanted the users to navigate to the page with images and see whether the image in question was present. However to our default the question was worded poorly. There are no images of jello shots on the website, however there is an image of jello shots being made. This lead to about 50% of the users answering yes, while the other half answered no. The live testers spent quite some time combing the images for signs of jello, but was quickly able to navigate to the site with the images. time In addition every person we saw live was able to figure out that clicking on the images zooms.


  • You did not just lose your dignity yesterday, but also your vallet and want it back. What is Realfagskjellerens email?

In order to answer this question the user had to click an image of a letter on the bottom in the footer of the page. This letter was only immediately visible on the pages with no scroll, as the footer sticks to the bottom. About 57% of the users were able to answer this question. Several live testers attempted to find the information in the "about us" page, but after some time they were able to correctly find the adress. At most this took 2 minutes and 3 missclicks.


  • To make yourself feel better you decide to post the embaressing photos you found of your mate. Which # does Realfagskjellerens use on instragram?

Similarly to the question above this one could be answered by clicking the instagram icon in the footer. About 70% of the users were able to answer this question. humorously one user finally admitted in their answer that they had yet to visit Realfagskjellerens page.


  • How easy was it to use the page to naviagate through the questions above?

We see that these questions were harder to answer than the previous ones. About 37% still thought the questions were easy, while 12% found them hard and 18% found them neither hard nor easy. One user in particular noted that finding the number of attendees from facebook were challenging.


General Feedback

  • How tempted are you in visiting Realfagskjelleren after using the page?

About 80% said that they wanted to visit Realfagskjelleren after using the page. About 6% said that they had gained some interest, while 12% said that they were ambvivalent.


The users who said their interest in Realfagskjelleren had not increased where asked why. Here the response where very varying. About 12% said that they obtained the same information from facebook, 15% did not attending events where alcohol were present, while 10% complained about the menu. When asked after the survey what the problem with the menu was the live tester explained about the small fontsize on his mobile


  • What do you think about the appereance of the site?

One user complimented us on the beautiful symmetry of the website. About 30% really loved the design, and said: "It feels just like being in Realfagskjelleren". Approximately 18% though the page looked fine, while 18% said it mediocre. The remaining users did not have any strong opinions either for or against the design.


  • Was there something in particular that caught your eye, and you liked about the site?

At the end of the survey we gave the users three questions they could answer freely. Several users complimented us on the games. One user liked the color combination (what?). It was also remarked that navigating the page was surprisingly easy, and everything was very organized.


  • Was there anything with the page that did not work or could be improved?

The users of Realfagskjelleren seems really into games, as again several users wanted more games. Some others complained that one could not order alcoholic beverages on the website. More seriously we recieved complaints about the following both in the form and from the livetesters:

  • The color choices for the active page and timer
  • Image placement on the About us page
  • The site does not work
  • There is no space between the text and the borders of the page
  • If you could add one additional feauture to the site, what would you add?

Again we recieved some quite humourusly responses. One user wanted to have a button to send a Taxi to Realfagskjelleren, while another wanted home delivery of beverages from Realfagskjelleren. More seriously we noted the following:


  • Multiplayer version of "Finn toeren", more functionality in the "dice" game.
  • More games, they wanted something called "Realfags ruletten"?
  • Number to Norwegian Poison Information Centre and National drug helpline in Norway
  • More student songs

Your Response

Deciding which responses to take seriously and which to ignore posed some difficulty. However with the help of live testers we where able to narrow down which changes where the most dire to improve.


  • The page does not work

    • As shown in the form nearly all of our users use Chrome, while the remaining handfull uses Firefox. This actually became a problem as we discovered our script for fetching events did not work in Internet Explorer, but more importantly Safari. Thus, it is our opinion that the most dire thing to fix is to make the website fully functional in Safari. We had spent about 1-2 days combing our Javascript files to find the error, alas we ran out of time for bug hinting and had to instead focus on finishing the reports. While it unnacceptable that the events and timer functionality does not work in Safari, it is not completely a disaster as mentioned previously nearly all users use a browser our site supports.


  • Image placement on the About us page

    • Previously we had two images on the "About us" page. This was changed about half way throught the survey, to just one image and text side by side. This coincidentally also fixes the problem with the text running into the margin.


  • The color choices for the active page and timer

    • The colors for realfagskjelleren are by default black and orange, and this makes adding a third color surprisingly difficult. We did try to add the complimentary color of orange - in other words blue - to our site. However, that looked even worse. Thus, we temporarly settled on green as it is a very natural color for something that is active. We totally forgot to change these colors before sending out the forms.


    One solution to this problem could be to add some shades of gray to the active menu, while changing the colors for the timer seems more difficult. One solution might be to talk with some of our friends on NTNU who studies some form of design and help us find good complimentary secondary colors.


  • There is no space between the text and the borders of the page. The textsize is very small.

    • Again we have no excuse for this. Adding padding and margins for paragraphs and headliners is very simple. We have just been lazy and had our focus elsewhere.


    Our group did have a short discussion about textsize though. We were unsure if we should scale the text, or if this was something that should be left ot the user. As our banners, footers are fixed increasing the fontsize does not affect the page design.


As a whole the users found the site to be very easy to navigate. While most users had difficulty finding the number of attendees this is mostly facebooks fault not ours. We talked with Andreas (the leader of Realfagskjelleren) and while we could have fetched the number of attendees and displayed that number on the webpage, this was not desirable. In addition that question was made in order to check whether the users were able to deduce that the event images were hyperlinks.
